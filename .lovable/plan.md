
## Concept

No email/password. Each workspace is a `workspace_id` (UUID). Devices join a workspace by entering a short **activation code** generated by an already-paired device. The first device creates the workspace on the fly. From then on, your phone and your PC see the exact same profiles, tasks, stores, proxies, and runner — because they're members of the same workspace.

Think of it like pairing a Plex/Steam Link client, not signing into a website.

## How pairing works

1. **First launch (no workspace yet)** — the app shows two buttons:
   - **Create new workspace** — generates a `workspace_id`, stores it + a per-device token in `localStorage`, you're in.
   - **Join existing workspace** — enter a 6-character code from another device.

2. **Adding a 2nd device (e.g. phone)**:
   - On your PC, open Settings → Devices → **Generate activation code**. A 6-char code appears (valid 10 min, single-use).
   - On your phone, open the app → **Join existing workspace** → enter the code.
   - Phone receives a device token bound to the same workspace and immediately syncs.

3. **Managing devices** — Settings → Devices lists every paired device with a friendly name (auto from user-agent, editable), last-seen timestamp, and a **Revoke** button. Revoking deletes that device's token; it gets booted on next request.

4. **Lost all devices?** — we show a one-time **recovery code** (long random string) at workspace creation, and again on demand from any active device. Anyone with the recovery code can re-pair. The user is responsible for saving it (we'll make this very explicit in the UI).

## Auth model

- No Supabase Auth, no email, no Google. Pure custom token auth.
- Each request from the browser sends `x-workspace-device-token` in headers.
- A server middleware (`requireWorkspaceDevice`) looks up the token, returns `{ workspaceId, deviceId }`, updates `last_seen_at`.
- All workspace-scoped tables (`checkout_profiles`, `tasks`, `stores`, `proxy_groups`, `app_settings`, `runner_devices`, `runner_jobs`, `runner_results`) get a `workspace_id` column. RLS stays disabled on these — server-only access via service role + middleware enforcement. (Same pattern the runner tables already use.)
- Runner pairing codes get scoped to a workspace too, so a runner only ever serves jobs for its workspace.

## Cloud sync (what moves to DB)

| Table | Replaces |
|---|---|
| `workspaces` | new |
| `workspace_devices` | new (this is the activation-code identity) |
| `workspace_activation_codes` | new (single-use 6-char codes) |
| `checkout_profiles` | `aio:profiles` + active list |
| `stores` | `aio:custom-stores` |
| `proxy_groups` | `aio:proxy-groups` |
| `tasks` | `aio:tasks` |
| `app_settings` (JSON blob) | misc localStorage keys |

Card numbers stay device-local — never written to DB. Profile editor labels these fields "Stored on this device only".

## One-time migration

On first paired session, if local `aio:*` data exists and the workspace is empty, push local → cloud, then keep local as a read-through mirror for offline.

## Technical Details

- New routes:
  - `pair.tsx` — public; "Create workspace" / "Join workspace" / "Recover".
  - `_paired.tsx` — pathless layout that checks `localStorage` for a device token and either renders `<Outlet />` or redirects to `/pair`.
  - Move current `index.tsx` under `_paired/index.tsx`.
- New server functions in `src/lib/workspace.functions.ts`:
  - `createWorkspace()` → `{ workspaceId, deviceToken, recoveryCode }`
  - `generateActivationCode()` (paired devices only)
  - `redeemActivationCode(code, deviceName)` → `{ workspaceId, deviceToken }`
  - `redeemRecoveryCode(code, deviceName)` → same
  - `listDevices()`, `revokeDevice(id)`, `renameDevice(id, name)`
  - All CRUD for profiles/stores/proxies/tasks/settings, scoped to the device's workspace.
- New middleware `src/integrations/workspace/middleware.ts` mirroring `requireSupabaseAuth`'s shape: reads `x-workspace-device-token`, looks up workspace, attaches `{ workspaceId, deviceId }` to context, updates heartbeat.
- New client attacher (like `attachSupabaseAuth`) that adds `x-workspace-device-token` from localStorage to every server-fn call. Registered in `src/start.ts` `functionMiddleware`.
- DB migration creates all new tables with `workspace_id uuid not null`, indexed; activation/recovery codes hashed at rest; `service_role`-only grants (no anon/authenticated). Adds `workspace_id` to existing `runner_*` tables and backfills any orphans to a fresh "legacy" workspace.
- New components: `AppShell.tsx` (bottom tab bar), `DevicesPanel.tsx`, `PairWorkspace.tsx`, `RecoveryCodeDialog.tsx`, `MigrationGate.tsx`.
- Root `__root.tsx` keeps QueryClient and gets a small "token changed" listener that invalidates queries when the device is revoked or re-paired.

## Out of scope (deferred)

- Per-device permission levels (e.g. "mobile read-only").
- Multi-workspace switching from one device.
- Analytics dashboard, task templates, webhook notifications, proxy health checker — still on the menu for later.
